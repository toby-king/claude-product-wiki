# Strategy Page Template

Use sparingly. Skip this page entirely if there's no substantive strategy work. An empty or vapid strategy page is worse than no strategy page.

## When to create this page

Only create when the user has actual strategic content to capture: defined target customer, clear positioning, deliberate trade-offs, articulated bets. If they don't, skip.

## Properties

- **Name**: `{Product Name} Strategy`
- **Owner**: User
- **Verification**: Empty (draft)
- **Tags**: `Product`, optionally `Overview`

## Icon and cover

- **Icon**: 🎯
- **Cover**: abstract / strategic-themed gallery image

## Body structure

### 1. Definition callout

```
ℹ️ This page captures the current strategic thinking for {product}. It will go stale — the date below is when it was last meaningfully reviewed.
```

### 2. Last reviewed line

> Last reviewed: {date}

### 3. Target customer (1 paragraph)

Specific. Who is the product for in concrete terms — not "businesses" but "UK SMEs in manufacturing/construction with 10-50 employees who need lead generation but can't afford a full BDR."

### 4. The problem (1 short paragraph)

What's the customer's actual problem in their words. What are they doing today instead.

### 5. The wedge (1-2 sentences)

The specific thing this product does better than alternatives. The reason a customer would pick it.

### 6. Deliberate non-goals (bulleted list)

What the product is deliberately *not* doing. Often more useful than the goals list because it forces clarity.

### 7. Key bets (bulleted list)

The strategic assumptions the product is built on. Each bet is something that could be wrong; if it is, the strategy needs to change.

Example format:
- We're betting that {X}, because {Y}. If wrong, we'd need to {Z}.

### 8. Open strategic questions (callout)

```
💡 Open strategic questions
- {question 1}
- {question 2}
```

## What NOT to include

- Detailed market analysis (lives elsewhere if at all)
- Financial projections
- Roadmap (deferred)
- Competitor matrix unless it directly informs positioning
- Anything aspirational that the user hasn't actually committed to
